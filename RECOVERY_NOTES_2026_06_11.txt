## IMPORTANT XFORMERS NOTE

xformers 0.0.31 caused import failures after the environment was upgraded to torch 2.8.0+cu128.

Observed error:

undefined symbol:
_ZN3c104cuda9SetDeviceEa

The following command resolved the import issue:

pip uninstall -y xformers

After removing xformers:

python -c "
from flash_head.inference import get_pipeline
print('IMPORT_OK')
"

produced:

IMPORT_OK

Note:
This does NOT mean all future xformers versions are incompatible.

It only confirms that the installed xformers build at the time was incompatible with the final torch installation.

## RECOVERY PROCEDURE

If recreating from scratch:

1. Create a new RunPod pod.
2. Clone repository.
3. Checkout tag:

git checkout SOULX_WORKING_RUNPOD_2026_06_11

4. Download model files:

   * SoulX-FlashHead-1_3B
   * wav2vec2-base-960h

5. Reinstall dependencies.

6. Verify package versions using:

   * WORKING_REQUIREMENTS_2026_06_11.txt
   * working_environment_WORKING_2026_06_11.txt
   * final_environment_snapshot_WORKING_2026_06_11.txt

7. If import errors reference xformers:

   * Verify torch version
   * Remove incompatible xformers installation

8. Run import test:

python -c "
from flash_head.inference import get_pipeline
print('IMPORT_OK')
"

9. Start FastAPI server.

10. Verify:

    * /ping
    * /pipeline-status

11. Test sample generation.
